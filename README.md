## 📖 О проекте

**OSINT P** — это компактное, но функциональное приложение для исследователей, журналистов, аналитиков безопасности и специалистов по расследованиям. Архитектура построена вокруг концепции **«Дело» (Case)**: каждое дело содержит независимый набор целей (персон), связей, категорий данных и доказательств.

Проект реализован как **single-file source** для удобства аудита кода, быстрого развёртывания и минимизации зависимостей. Все данные хранятся **исключительно локально** на устройстве пользователя. Никакой телеметрии, облачных синхронизаций, внешних API или скрытой передачи данных.

## ✨ Ключевые возможности

### 🗂 Управление делами и целями
- Создание неограниченного количества кейсов с гибким названием и тегами.
- Добавление целей (персон): ФИО, заметки, теги, приоритет (High/Medium/Low).
- Быстрая смена приоритета прямо из списка карточек (экспериментально).

### 🧩 Гибкая структура данных
- Вложенные **категории** с произвольным названием.
- Ключ-значение пары внутри категорий с возможностью перетаскивания (drag & drop reorder).
- Автоматическое определение типов значений:
  - 📞 Номера телефонов
  - 💳 Банковские карты (13–19 цифр)
  - 🌍 Геокоординаты (`lat, lng`)
  - 🏷 Пользовательские маркеры (настраиваются в настройках)
- Кнопки копирования и открытия на карте появляются автоматически при обнаружении паттерна.

### 🔍 Умный поиск
- Fuzzy-search с алгоритмом Левенштейна.
- Поиск по всем полям, категориям, тегам, заметкам и причинам связей.
- Подсветка найденных фрагментов с контекстом (`...snippet...`).
### 🔗 Граф связей
- Визуализация связей между целями внутри дела.
- Кастомный **force-directed layout** с предотвращением наложения узлов, 500+ итераций симуляции.
- Фильтрация рёбер по причинам связей.
- Экспорт графа в PNG (с шарингом/сохранением).

### 📍 Карты
- Интеграция `flutter_map` + OpenStreetMap тайлов.
- Персональная карта: отображение всех координат из категорий цели.
- Глобальная карта дела: маркеры всех целей с кластеризацией и подписями.

### 📎 Доказательства (Evidence)
- Прикрепление файлов (изображения, документы) к любой цели.
- Автоматическое копирование в директорию приложения.
- Встроенный просмотр изображений и открытие файлов через системный viewer.

### 📄 Генерация PDF
- Экспорт отчёта по отдельной цели или по всему делу.
- Настройки: включить/исключить связи и доказательства.
- Поддержка Google Noto Sans, автоматическая разбивка на страницы A4.
- Предпросмотр, печать, сохранение и шаринг.

### 🎨 Интерфейс и локализация
- Material 3 дизайн, адаптивная верстка.
- 3 темы: `Light`, `Dark`, `Pure AMOLED`.
- Свайп/долгое нажатие для быстрого переименования/удаления.

## 🧭 Руководство пользователя

1. **Главный экран** — список дел. Нажмите `+` чтобы создать новое. Поисковая строка ищет по всем вложенным данным.
2. **Внутри дела** — список целей. Долгое нажатие на карточку открывает меню: переименовать, добавить теги, удалить.
3. **Карточка цели** — 4 вкладки:
   - 📋 **Инфо**: категории, ключи-значения, перетаскивание для сортировки.
   - 🔗 **Связи**: выбор другой цели из дела, указание причин (каждая с новой строки).
   - 📎 **Доказательства**: выбор файлов, прикрепление к цели.
   - 🌍 **Карта**: маркеры всех найденных координат в категориях цели.
4. **Шапка дела** — кнопки: `🗺 Глобальная карта`, `📄 PDF дела`, `🔗 Граф связей`, `⚙️ Настройки`.
5. **Настройки** — переключение темы/языка, управление кастомными марками копирования, импорт/экспорт JSON/SQLite, сброс базы, тоглы экспериментальных функций.

## 🧪 Экспериментальные функции

Включаются в `Настройки → Experimental Features`. Требуют ручного активирования, так как находятся в стадии стабилизации:
| Функция | Описание |
|--------|----------|
| `exp_priority` | Бейджи приоритета (High/Med/Low) на карточках целей + быстрый выбор из списка |
| `exp_global_map` | Единая карта дела с маркерами всех целей |

| `exp_case_tags` | Добавление и редактирование тегов на уровне дела |
| `exp_case_pdf` | Генерация PDF-отчёта по всему делу (все цели, связи, доказательства) |

## 🔒 Безопасность и приватность

- ✅ **100% Offline**: приложение не отправляет и не получает данные по сети.
- 💾 **Локальное хранение**: JSON-файл + опциональный SQLite дамп в `ApplicationDocumentsDirectory`.
- 🗑 **Сброс данных**: функция `Reset database` полностью очищает все кейсы, цели и файлы доказательств.
- 📜 **Правовое требование**: разработчик явно указывает: *«Используйте законно»*. Приложение не обходит законы о защите данных, не собирает телеметрию и не предполагает несанкционированный доступ к информации.

## 📐 Архитектура кода

Несмотря на формат single-file, проект разделён на логические блоки:
- `AppState` — singleton с управлением состоянием, сериализацией JSON/SQLite и настройками.
- `Models` — строго типизированные классы (`CaseFile`, `Person`, `CategoryBlock`, `ConnectionLink`, `EvidenceItem`, `Settings`).
- `Parsers / Detection` — `ValueDetector` с регулярными выражениями для телефонов, карт, координат.
- `FuzzySearch` — реализация расстояния Левенштейна + word-level matching.
- `Graph Engine` — кастомный `_ForceDirectedGraphView` с физической симуляцией отталкивания/притяжения и разрешением наложений.
- `PDF Builders` — изолированные генераторы `PdfBuilder` и `CasePdfBuilder` с поддержкой изображений и многостраничной вёрстки.
